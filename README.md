# ⚙️ Agentic OS - 31 Harbor Agency Dashboard

> An AI-powered workflow management and automation hub, inspired by the Agentic OS concept, designed to orchestrate tasks and provide observability across various business domains for 31 Harbor, Party Favor Photo, and XMRT DAO.

## Overview

This dashboard has been upgraded to an **Agentic OS** interface, transforming from a simple email operations tracker into a comprehensive workflow management system. It visualizes the core components of an AI-driven agency, including:

-   **The Conductor**: The central AI orchestrator (conceptualized as "Claude Code").
-   **Domain-Specific Workflows**: Organized into Research, Content, Community, and Agency domains, each with specific "Skill Cards" for task execution.
-   **Automation Layer**: Illustrates the integration of various tools and software assets (GitHub, Slack, Email, Obsidian, Resend, Cloudflare, Analytics, Auth).
-   **Memory Vault**: A structured representation of an Obsidian-based knowledge base, categorized into Raw data, Wiki (processed reports), and Output (deliverables).
-   **Observability Dashboard**: Provides real-time metrics on agentic activity, API usage, active routines, and success rates, along with quick skill execution buttons.
-   **Company Operations Overview**: Summarizes key metrics for 31 Harbor, Party Favor Photo, and XMRT DAO, including email statistics and DNS verification status.
-   **Activity Log**: Displays recent system activities and email events.

## Features

-   **Modular Design**: Easily expandable with new domains and skill cards.
-   **Real-time Data Integration**: Fetches and displays live data from `data.json` (generated by `scripts/update_stats.py`).
-   **Responsive UI**: Optimized for various screen sizes.
-   **Dark Theme**: Modern and professional aesthetic.

## Structure

-   `index.html`: The main dashboard interface, implementing the Agentic OS design.
-   `data.json`: Stores dynamic data for the dashboard, updated by `scripts/update_stats.py`.
-   `scripts/update_stats.py`: Python script responsible for fetching data (e.g., from Resend API) and updating `data.json`.
-   `CNAME`: Custom domain configuration for GitHub Pages.
-   `.github/workflows/deploy.yml`: GitHub Actions workflow for deployment to GitHub Pages.

## Setup & Deployment

This dashboard is deployed via GitHub Pages. The `deploy.yml` workflow handles automatic deployment on pushes to the `main` branch. The custom domain `agency.31harbor.com` is configured via the `CNAME` file.

To update the dynamic data, ensure `scripts/update_stats.py` is run periodically to refresh `data.json`.
